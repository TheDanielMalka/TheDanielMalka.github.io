About

Documented Delivery.
Not Tutorial Tourism.

I run a public DevOps roadmap like production R&D: Jira sprints, GitHub milestones, smart commits, PRs that close issues, and Actions that lint and test real code — starting from a structured monorepo (Node, Express, middleware, JSON APIs) and professional dependency tooling.

On Linux I went deep on OS fundamentals, shell pipelines, FHS and permissions, log paths (journalctl, rotation), and built a Bash resource monitor with Slack webhooks, cron, logrotate, and idempotent setup.sh — the same class of automation you ship beside apps.

Docker is end-to-end: engine mental model, run/inspect/exec, images and layer cache, bridge networking and DNS between containers, named volumes vs bind mounts, Dockerfiles, Compose, registry login/tag/push — plus Compose + nginx + Postgres on EC2 behind security groups.

AWS tracks IAM (MFA, least-privilege experiments), custom VPC with public/private subnets, IGW and routes, EC2 bootstrap (SSH keys, Docker, Python venv, repo clone), S3 policies and versioning, static hosting concepts, and CLI workflows including IAM roles on EC2 instead of long-lived keys where it matters.

When hardware failed, I kept momentum on Termux, then rebuilt Windows + WSL, restored GitHub/Jira sync, and validated the same workflow across machines — zero-downtime learning as a habit, not a slogan.
